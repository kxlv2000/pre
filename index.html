混合 YOLOv11 × Qwen3-VL · 实时性

  • 混合架构：L1 侧用 YOLOv11 级检测器做人脸框、关键点与简单规则，高帧率、低算力；L2 侧用 Qwen3-VL 做语义推理（疲劳、分心、危险动作等），低频触发。

硕士毕设：MSc in Artificial Intelligence（NTU）

AIGC · 自动驾驶异常数据增强 · Final Sharing

面向自动驾驶感知与仿真：利用生成式模型对稀有、危险或难采集场景做数据增强，缓解长尾与标注成本问题；与 CARLA 仿真及感知管线结合。

Stable Diffusion LoRA ControlNet Segment Anything CARLA GAN / Diffusion

方法与实验

  • 生成主干：以 Stable Diffusion 为核心，配合 LoRA 做领域适配；用 ControlNet（Canny / 深度等） 把道路几何与障碍物轮廓编码为条件，缓解纯 Text2Img 的不可控与伪影。
  • 区域约束：Segment Anything（SAM） 或同类分割给出可编辑掩码，只在关心区域内做增强，减少对背景建筑的无关改动，便于下游感知一致性检查。
  • 仿真数据：在 CARLA 中采集多天气 / 多光照的 BEV / 前视 序列，对稀有类别（如 Cyclist）做实例级增广，并与标注格式对齐以便回灌训练。
  • 路线对比：对比 Img2Img 引导与仅 Text2Img 在可控性、细节质量、训练稳定性上的差异；分享稿中给出同 prompt 可视化与失败样例分析。
  • 评价思路：除主观观感外，关注 FID / LPIPS 等图像指标与下游检测器AP 变化（若管线允许），避免「生成好看但感知变坏」。

合作课题：校企 · 生成式世界模型

德赛 × NTU / 中大 · DriveArena 类双核

目标：高保真交通场景仿真与生成数据，提升感知并支撑闭环测试；周期约 6 个月，分月检查点。验收侧：德赛数据集 3D 检测 / 车道线「仅生成数据」平均提升 约 4%；NuScenes Occ 任务平均提升 约 10% 以体现跨数据集泛化。

Traffic Manager Diffusion 渲染 OSM BEV 长尾场景 LLM

架构与分工

  • 双核思路：交通管理器（Traffic Manager） 负责 OSM / 路网、动静态参与者与交互逻辑，为生成提供结构化条件；扩散渲染器 负责 多视角 RGB 与 BEV 的高保真输出，贴近 DriveArena 类方案。
  • 与端到端自驾衔接：世界模型作为闭环验证的一环，接收控车信号预测未来帧，用于碰撞率、场景覆盖率等指标，而非单点「好看视频」demo。

技术要点

  • BEV 预标：在 GeMap / VMA 路线下做车道与底图要素检测；自定义 Point2Line 等损失拉近预测折线与真值拓扑，兼顾Chamfer 类指标与业务可解释性。
  • 流式增广：对超大分辨率底图（16k×8k 量级）采用 2D 裁剪流式读取，在同等 batch 下显存约 −40%，支撑更长训练迭代。
  • 分布式训练：DeepSpeed 多卡流水线、梯度累积与混合精度；分别跟踪 车道线 / 道路边界 / 停止线 等子任务的验证 mAP 曲线，定位过拟合 / 域偏移。
  • 多传感器：RGB + LiDAR 强度 融合实验，表格化输出 Recall / Precision / F1@阈值，对比仅视觉与融合在雨雾等场景的增益。
  • 交付习惯：实验记录 PDF + Weekly Update 固定超参、数据版本、指标表，便于半年后回溯。
